Angular Lifecycle

--Lifecycle hooks are special method provided by angular that allows us to tap into different stages of a component's lifecycle. From the moment a component is created to the moment it is destroyed, Angular provides hooks that let us run custom logic at specific times. These hooks can be used to perform tasks like setting up the component, fetching data, or cleaning up resources.

--We can divide angular life cycle hooks in 3 stages as flollows:

  • Initialization :
    Some time we need to set-up/fetch some data or state as soon as the component is created.
  • Change Detection:
    We might need to perfome actions when data-bound properties change.
  • Cleanup:
    It's important to clean up resources or subscriptions to avoid memory leaks when the component is destroyed.

    • Here is a lsit of some angular lifecycle hooks

    • ngOnInit : Called once the component is initialized.
    • ngOnChanges : Called every time the input properties of a component changes.
    • ngDoCheck : Called during every change detection run
    • ngAfterContentInit: Called once after the component's content has been initialized.
    • ngAfterContentChecked:Called after every check of the component's content.
    • ngAfterViewInit: Called once after the component's view (and child views) has been initialized.
    • ngAfterViewChecked: Called after every check of the component's view (and child views).
    • ngOnDestroy: Called just before the component is destroyed
--ngComponentOutlet is a directive that allows us to dynamically load a component
